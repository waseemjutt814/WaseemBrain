
Industrial Hardening Plan for Lattice Brain
Summary
Rebuild the repo into a real local AI stack for CPU-only, 8-16 GB RAM, Windows + Linux, with ultra-fast text serving and no runtime placeholders. The implementation order will be: remove fake/demo paths, replace persistence and routing with real low-RAM components, convert experts into real grounded executors, fix interface/bridge concurrency and multimodal APIs, then update repo skills and validation so the project cannot regress back into stub/demo behavior.

Implementation Changes
Lock runtime into strict real mode
Remove production-path stubs and placeholder behaviors from runtime code: stub gateway bootstrap, static keyword router fallback, echo-style expert fallback, tiny-onnx expert generation, and fake learning artifacts.
Keep test doubles only inside tests/; runtime startup must fail fast if required expert/router/index assets are missing or invalid.
Split dependencies into runtime, training, and dev groups. Runtime keeps only low-RAM serving deps; heavy trainer/export deps move out of default boot/install.
Add a repo guard that fails CI if runtime source references demo/stub modes, or if checked-in production expert/router artifacts are obviously placeholder-sized or placeholder-typed.
Replace fake memory backends with a real low-RAM memory engine
Replace JSON-vector fallback and XML/GraphML rewrite backends with a single real memory stack: SQLite metadata + session graph edges + HNSW vector index.
Use all-MiniLM-L6-v2 as the default embedding model, exported/served through ONNX Runtime with CPU-friendly quantized weights.
Store node metadata, provenance, session links, access stats, and graph edges in SQLite; store vectors in the HNSW index and enough metadata for rescoring.
Remove hardcoded recall values; honor settings-driven limits and thresholds everywhere.
Stop storing uncited generated answers as high-confidence recallable truth. Memory nodes must carry provenance and source category; recalled answers must be evidence-backed.
Replace keyword routing and fake experts with real grounded executors
Replace the Rust router heuristic with a trained hashed TF-IDF + logistic regression router. Training data lives in versioned curated JSONL files; training/export happens in Python; Rust loads a compact weights artifact and performs real scoring at runtime.
Keep the current expert IDs but change their implementation from fake ONNX echo files to real executors:
language-en: evidence-backed answer composer over memory and optional web evidence.
code-general: workspace-aware code retrieval over repo files plus snippet-cited answer synthesis.
geography: offline dataset-backed exact lookup using a permissively licensed static country/capital dataset.
Change expert loading from “every expert is an ONNX file” to “expert is a typed executor with optional artifacts/indexes”. Remove full-file byte copying and use bounded async execution with LRU eviction.
If an expert cannot answer, return an explicit unsupported/error result; never echo user input as an answer.
Fix the serving surface and bridge
Keep existing POST /query working as a text alias for backward compatibility.
Add real staged-compatible HTTP endpoints:
POST /query/text with JSON body.
POST /query/url with JSON body.
POST /query/file with multipart upload.
POST /query/voice with multipart upload for WAV/PCM.
Keep WebSocket streaming for text and url first-class in this pass; do not add WS binary uploads until HTTP multimodal paths are stable.
Replace the Python bridge single-flight loop with concurrent request task handling, cancellation, backpressure, and independent health/metadata requests.
Extend health/reporting so the server exposes component readiness, loaded models/indexes, resolved backends, and per-subsystem latency/error counters.
Make internet and learning real
Keep internet retrieval as local-memory-first and optional enrichment only.
Replace blocking search/rate-limit behavior with async-safe throttling and bounded fetch concurrency.
Keep feedback collection and scoring, but remove fake auto-correction output generation from runtime. “Learning” in this pass means real logging, scoring, and offline job preparation only.
Disable any correction/export path that cannot produce a real trained artifact yet.
Update repo skills to match the real architecture
Keep existing project skills but rewrite them around the hardened architecture and no-placeholder rules.
Add missing project skills for router-daemon, interface-bridge, and quality-gates.
Update scripts/sync_codex_skills.py so generated skills reflect the real subsystem order and current acceptance rules, not the earlier greenfield/demo framing.
Keep SKILL bodies short, move detailed architecture and checklists into references/, and extend skill validation so generated skills must include anti-placeholder policy, real runtime expectations, and current subsystem references.
Update create_expert.py so it creates real expert manifests/artifact bundles only; it must never generate demo ONNX placeholders again.
Important API / Type / Contract Changes
QueryRequest becomes a discriminated union by modality; the current text request shape remains supported as a compatibility alias.
ExpertMeta gains typed execution fields such as kind, artifacts, capabilities, and load_strategy; expert definitions are no longer “file-only ONNX entries”.
ExpertOutput gains evidence/citation fields so assembled answers are grounded.
MemoryNode gains provenance/source-reference metadata and cannot be treated as trusted recall without backing evidence.
Router artifact format changes from placeholder ONNX pathing to a real trained weights artifact consumed by Rust.
Health responses expand to include readiness and backend/model/index status, not just counts.
Test Plan
Startup fails if required router/expert/index assets are missing; no runtime stub path remains.
Router training/export produces a real artifact and Rust inference passes curated holdout tests without keyword shortcuts.
Warm text-query target is <= 1s for simple queries and cold text-serving startup target is <= 2s on the selected CPU profile; p95 regression tests run in CI-friendly benchmarks.
Concurrent query, health, memory, and experts calls do not block each other through the bridge.
text, url, file, and voice HTTP routes work end-to-end with real adapters; voice uses Whisper base.en int8 via faster-whisper.
Memory persistence/retrieval remains correct under load, does not rewrite full state per mutation, and respects session/provenance rules.
Expert responses include evidence; unsupported queries fail explicitly instead of returning echoed input.
Internet module rate limiting does not block the event loop and respects allowlist/freshness rules.
Skills sync and validation fail if source/generated skill content drifts or if placeholder/demo guidance reappears.
Full validation runs on Windows and Linux across Python, TypeScript, and Rust.
Assumptions and Defaults
Runtime target is CPU-only, 8-16 GB RAM, Windows + Linux, permissive-license components only.
Architecture is retrieval + small real experts, not a large central local LLM.
Internet remains optional and off the critical path for most queries.
Learning remains feedback/logging only until a real trainer/export pipeline is ready.
Deterministic heuristics are allowed only when they are the intended real algorithm; fake/demo/stub behavior is not allowed anywhere in runtime code or production assets.
